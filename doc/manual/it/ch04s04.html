4.4. Preparazione dei file per l'avvio da disco fisso

L'installatore può essere fatto partire usando i file d'avvio scritti su una partizione già esistente del disco fisso, lanciandoli da un altro sistema operativo o richiamando un bootloader direttamente dal BIOS. Sui moderni sistemi UEFI il kernel può essere avviato direttamente dalla partizione UEFI senza la necessità di un bootloader.

Usando questo metodo è possibile realizzare una installazione completa puramente da rete evitando tutte le noie dei supporti rimovibili come la ricerca e la masterizzazione delle immagini dei CD/DVD.

4.4.1. Avvio da Linux dell'installatore da disco fisso con GRUB

Questa sezione spiega come aggiungere o sostituire una installazione di linux esistente usando GRUB.

All'avvio GRUB supporta il caricamento in memoria non solo del kernel ma anche di un'immagine del disco. Questo RAM disk può essere usato come filesystem di root dal kernel.

Copiare i seguenti file dagli archivi Debian in una directory del proprio disco fisso, per esempio /boot/newinstall/.

  • vmlinuz (kernel)

  • initrd.gz (immagine del ramdisk)

Se si vuole usare il disco fisso solo per l'avvio e poi scaricare tutto il resto dalla rete si dovrebbe scaricare il file netboot/debian-installer/amd64/initrd.gz e il relativo kernel netboot/debian-installer/amd64/linux. Questo permette di rifare le partizioni sul disco fisso da cui è stato avviato l'installatore, anche se questa operazione deve essere fatta con molta attenzione.

In alternativa, se si vuole lasciare intatta una partizione già esistente sul disco, si può scaricare il file hd-media/initrd.gz, il relativo kernel hd-media/vmlinuz e anche una copia dell'immagine per l'installazione sul disco (assicurarsi che il nome termini in .iso). In questo modo l'installatore si avvia dal disco e l'installazione avviene usando l'immagine, senza la necessità della rete.

Infine, per configurare il bootloader procedere con Sezione 5.1.5, «Avvio di Linux con GRUB».

4.4.2. Avvio da DOS dell'installatore da disco fisso con loadlin

In questa sezione è spiegato come preparare il proprio disco fisso per avviare l'installatore da DOS usando loadlin.

Copiare le seguenti directory dall'immagine per l'installazione Debian in c:\.

  • /install.amd (binari del kernel e immagine del ramdisk)

  • /tools (programma loadlin)
